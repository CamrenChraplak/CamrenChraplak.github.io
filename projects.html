Musical Robot Arm

(10/2024 to 12/2024)

 "The Arm” is a five degree of freedom (DOF) robot arm that is able to play a small MIDI keyboard and various percussion instruments, such as a drum and a cymbal. It is capable of playing up to two notes simultaneously on the keyboard with its linear gripper, and it plays the drum with a wooden drumstick mounted flexibly on its end-effector. It parses input in MIDI format, and is capable of playing one instrument at a time. “The Arm” is able to play quarter notes at a maximum of 120 BPM, though its piano playing accuracy does decrease with play rate. To achieve proper timing, the MIDI file is pre-processed before playing into a joint-space trajectory which ensures that the end-effector intersects the desires positions at precisely the note-on times read from the MIDI input."

Open Source Desktop CNC Mill Senior Project

(8/2024 to Present)

 Me and my team are expanding on a previous team's design for a desktop CNC mill. For improving current systems, we will construct a more robust outer frame, design a better coolant sealing/draining, have more user friendly software, and improve the safety of the machine. In addition, we will integrate new features such as auto bed leveling, a 4th axis, tool changer, and integrated computer interface. Our base model mill will cost under $2,500 with optional addons that allow the user to upgrade their machine. This system will be easy to manufacture, fit in a 2ft by 2ft by 3ft, and be open source for hobbyists to recreate.


My overall goals for this project is to improve the current machine by

  • selecting a more user friendly software.
  • selecting a control board compatible with the new software.
  • implimenting tool changing, auto bed leveling, and machine monitoring through software.


The status of my goals for the project include

  • choosing Universal GCode Sender as the user interface.
  • selecting PicoCNC for driving the mainboard logic using grblHAL.
  • implimenting monitoring through accelerometers and temperature sensors.


The goals that are still a work in progress include

  • fully implimenting machine monitoring through strain gauges, a tachometer, and camera logging.
  • creating plugins for controlling an automatic tool changer and auto bed leveling at the hardware and user interface level.

Software Engineering Hospital Kiosk

(1/2024 to 3/2024)

 The goal of this project was to develop a kiosk application for Brigham and Women's Hospital. Several kiosks will be laid out across the hospital providing assistance to hospital patients, visitors, and employees. The application helps users find rooms and other points of interest in the hospital through an interactive map. Additionally, the application allows users to submit service requests for patient transportation, gift or medicine delivery, maintenance, and more. These services can be requested from any kiosk and carried out to any hospital location as required. The application also features admin controls to allow hospital personnel to edit the interactive map, import or export data from the database, view service requests and employee data, and more. The project sought out to create an application that was easy to use and intuitive in a kiosk setting. The hope is that the application will aid in customer service and improve customer satisfaction as well as aid hospital personnel in providing exceptional care for their patients.


My contributions to the project include

  • designing map interface. This includes navigation buttons, touch screen interface, resizing actions, node details, and floor changes.
  • keeping 11 person team on track with project goals and leading user stories as product owner.

Microcontroller Oscilloscope

(6/2023 to Present)

 Oscilloscopes in the modern age can cost hundreds of dollars for a basic model while it is often a necessity for any complex electronics project. To solve this, many tinkerers have developed their own oscilloscopes, with many posted online. However, most solutions involve multiple and specialized components, while lacking many functions of normal oscilloscopes. The goal of the project is to provide access to an oscilloscope, with the only necessary hardware being a micro-controller and a phone/computer, along with providing the functionality of most common oscilloscopes.


My overall goals for this project is to create an accessible oscilloscope by

  • having makers use microcontrollers they have on hand at a cheap price.
  • providing Wifi, Bluetooth, and Serial connections for various circumstances.
  • include addons for extra features (ie. external ADC's, AC voltage, etc.).


The status of my goals for the project include

  • protoyping preliminary web interface.
  • protoyping preliminary code for ESP32.
  • template for collecting and sending data to web page.


The goals that are still a work in progress include

  • fully implimenting a web interface for ESP32.
  • creating a HAL for use by mutliple microcontrollers.
  • creating apps for use by Bluetooth and Serial on PC.
